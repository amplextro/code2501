# Proposal: チャンク読み（Chunked Reading）

**Status**: Rejected
**Target**: v0.4

## 概要

長文バーコードをチャンク単位で順次読み取る機能。
XRグラス等の限られた視野でも、バーコードの途中からチャンク単位で独立してデコードでき、
最終的に全チャンクを結合して完全なテキストを復元する。

ストリーミング（逐次デコード）はECCブロック境界の制約で困難だが、
チャンク読み（独立デコード可能なブロック単位の読み取り）は実現可能。

## チャンクとECCブロックの関係

- **ECC Interval**: 既存パラメータ。N文字ごとにECCグリフを挿入（誤り訂正の単位）
- **Chunk Interval**: 新規パラメータ。M個のECCブロックをまとめて1チャンク（ナビゲーションの単位）
- チャンクはECCブロックの倍数で構成 → ECC境界と必ず整合

### バーコード全体構造（チャンクモード時）

```
ECC Interval = 10, Chunk Interval = 3 (ECCブロック数) の場合:

START | CB(0) [CHAR×10 ECC] [CHAR×10 ECC] [CHAR×10 ECC] | CB(1) [...] | ... | STOP
       └──────────────── chunk 0 (30文字) ──────────────┘
```

- CB(i) = CHUNK_BOUNDARY グリフ（チャンク i の先頭）
- 各チャンク内のECCグリフはそのチャンク内の文字のみをカバー
- CHUNK_BOUNDARYの挿入頻度を抑えてオーバーヘッドを削減

### オーバーヘッド例

| ECC Interval | Chunk Interval | チャンクあたり文字数 | オーバーヘッド |
|---|---|---|---|
| 10 | 3 | 30 | 約6.7% |
| 10 | 5 | 50 | 約4.0% |
| 20 | 3 | 60 | 約3.3% |

## CHUNK_BOUNDARY グリフ（新規）

### 基本仕様

| 項目 | 値 |
|---|---|
| 横幅 | 26列（START/STOPと同一 = CHAR 2グリフ分） |
| PUA | U+E003 |
| 役割 | チャンク境界マーカー + メタデータ + 独立Finder |

### レイアウト

```
列0-1  : 同期バー
列2-9  : Chunk Finderパターン（8列 × 16行）
列10-25: チャンクメタデータ（16列 × 16行 = 256bit）
```

### Chunk Finderパターン

START（ボックス型）やSTOP（ボックス反転型）とは異なる**水平ストライプ型**パターン。
デコーダがバーコードの中間部でも直接検出可能。

```
上8行: ████████   (全黒)
       ________   (全白)
       ████████   (全黒)
       ________   (全白)
       ██    ██   (両端マーカー: 方向判定用)
       ██    ██
       ________   (空白)
       ▀▄▀▄▀▄▀▄   (クロック)

下8行: 上記の反転（上下判別用、START/STOPと同じ慣例）
```

判別方法:
- START Finder: ボックス型（枠+内部正方形）
- STOP Finder: STARTの左右反転
- CHUNK Finder: 水平ストライプ型

### チャンクメタデータ（256bit）

| フィールド | ビット数 | 説明 |
|---|---|---|
| Marker Type | 2 | `10` = CHUNK_BOUNDARY（将来の他マーカー種別と区別） |
| Chunk Index | 14 | 0始まりのチャンク番号（最大16,383） |
| Total Chunks | 14 | 全チャンク数（デコーダが完了判定に使用） |
| Chars in Chunk | 6 | このチャンク内の文字数（最大63、ECC Interval × Chunk Intervalと一致） |
| Cumulative Offset | 16 | テキスト先頭からの文字オフセット（最大65,535） |
| Content Hash | 32 | チャンク内UTF-8バイト列のCRC-32（チャンク単位の整合性検証） |
| Flags | 8 | ビットフラグ（下記参照） |
| Reserved | 36 | 将来拡張用 |
| Metadata ECC | 128 | メタデータ自体のRS誤り訂正（GF(2⁸)） |

#### Flags ビットフィールド

| ビット | 意味 |
|---|---|
| 0 | `is_first` — 先頭チャンク（chunk index 0） |
| 1 | `is_last` — 末尾チャンク |
| 2-7 | Reserved |

## STARTヘッダの変更

現在の116 reserved bitsを以下のように割り当てる:

### 現行ヘッダ（v0.3）

| フィールド | ビット数 |
|---|---|
| Version | 4 |
| ECC Level | 2 |
| ECC Interval | 6 |
| Reserved | 116 |
| Header ECC | 128 |
| **合計** | **256** |

### 提案ヘッダ（v0.4）

| フィールド | ビット数 | 説明 |
|---|---|---|
| Version | 4 | 仕様バージョン（v0.4 = 1） |
| ECC Level | 2 | L(00) / M(01) / Q(10) / H(11) |
| ECC Interval | 6 | ECCグリフの挿入間隔（文字数, 1〜63） |
| Chunk Mode | 2 | `00`=チャンクなし（v0.3互換）, `01`=チャンクあり |
| Chunk Interval | 6 | チャンクあたりのECCブロック数（1〜63） |
| Total Chunks | 14 | 全チャンク数 |
| Total Characters | 16 | テキスト全体の文字数（最大65,535） |
| Global Hash | 32 | テキスト全体のCRC-32（全チャンク結合後の検証用） |
| Reserved | 46 | 将来拡張用 |
| Header ECC | 128 | ヘッダ自体のRS誤り訂正 |
| **合計** | **256** |

検算: 4 + 2 + 6 + 2 + 6 + 14 + 16 + 32 + 46 + 128 = 256 ✓

## PUA マッピング（追加）

| Unicode | グリフ種別 |
|---|---|
| U+E000 | START |
| U+E001 | STOP |
| U+E002 | ECC |
| U+E003 | CHUNK_BOUNDARY |

## デコーダの読み取りフロー（チャンクモード）

### 基本フロー

```
1. 可視領域をスキャン
2. Finderパターンを検出（START / STOP / CHUNK_BOUNDARY のいずれか）
3. 回転・スケール・歪み補正
4. CHUNK_BOUNDARYを検出した場合:
   a. メタデータ読み取り（chunk_index, total_chunks, offset等）
   b. 後続のCHAR + ECCグリフをチャンク内で復号
   c. ECCブロック単位で誤り訂正を実行
   d. Content Hashでチャンク整合性を検証
   e. バッファに格納（chunk_indexをキーに）
5. 全チャンク（0 〜 total_chunks-1）が揃ったら:
   a. chunk_index順に結合
   b. Global Hash（STARTヘッダ内）で全体検証
   c. UTF-8文字列として出力
6. 未取得チャンクがある場合:
   a. 取得済みチャンクのindexからギャップを特定
   b. ユーザーにスキャン方向を案内（「左へ」「右へ」等）
```

### エッジケース

**STARTグリフが見えた場合**:
ヘッダからchunk mode, total chunks, global hashを取得。
直後のCB(0)からchunk 0の読み取りを開始。

**STOPグリフが見えた場合**:
最終チャンクの末尾。左方向にスキャンして最後のCHUNK_BOUNDARYを探す。

**CHUNK_BOUNDARYが視野内にない場合**:
同期バーとグリフ構造から個別CHARグリフは復号可能（per-glyph ECC）。
ただしチャンク内位置が不明なため、左右にパンしてCHUNK_BOUNDARYを探す。

**重複スキャン**:
chunk_indexで重複を検出。同一チャンクを複数回読んでも問題なし。

## チャンクとハードブレーク/ソフトブレークの関係

- **ハードブレーク（`\n`）**: CHARグリフとして通常通りエンコード。チャンク境界とは無関係。1つのチャンク内に複数の改行が含まれることがある。
- **ソフトブレーク（表示折り返し）**: 表示上の折り返しのみ。チャンクとは独立。1つのチャンクが複数の表示行にまたがることがある。
- **CHUNK_BOUNDARYグリフ**: 表示折り返し時はSTART/STOPと同様に扱う（26w）。

## 後方互換性

| デコーダ | バーコード | 動作 |
|---|---|---|
| v0.3 | v0.3（チャンクなし） | 正常動作 |
| v0.4 | v0.3（チャンクなし） | Chunk Mode=00 → 単一チャンクとして正常動作 |
| v0.4 | v0.4（チャンクあり） | チャンク読み対応 |
| v0.3 | v0.4（チャンクあり） | Versionフィールドで非対応を検出・警告 |

## グリフ種別一覧（改訂）

| 種別 | 用途 | 横幅 |
|---|---|---|
| START | 開始Finder + ヘッダ情報 | 26列（固定） |
| CHAR | 1文字のデータ | 13列（固定） |
| ECC | 誤り訂正ブロック | 13列（固定） |
| CHUNK_BOUNDARY | チャンク境界 + メタデータ | 26列（固定） |
| STOP | 終端Finder + 全体チェックサム | 26列（固定） |
